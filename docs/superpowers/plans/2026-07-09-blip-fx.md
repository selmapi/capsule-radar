# Blip Enhancements — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** Per-theme blip enhancements — ATC leader line + heading-vector on Military/CIC/ClaudeIC, altitude-scaled size on Mission Control, plus global fade-in + pulse (new/emergency) on all themes.

**Architecture:** A `blipFx` bitfield on `ThemeDesc`; per-aircraft first-seen tracking for the temporal effects; all draws in `ac_draw_cb`.

**Tech Stack:** C++/LVGL v8, PlatformIO. Verify by compiling both envs; these render in the SDL sim but the window isn't observable here, so functional acceptance is on-device (Task 5).

**Ground rules:** branch `feature/blip-fx`; commit per task; don't push (flash/redeploy at end). Existing themes with `blipFx==0` and fully-faded aircraft (age ≥ FADEIN_MS) must render byte-identically to today.

## Task 3: Fade-in opacity + altitude-size scale threaded through the glyph

**Files:** `src/radar_view.cpp` (blip-shape helpers + `ac_draw_cb`)

- [ ] **Step 1 — Named constants** at the top of the aircraft-draw section:
```cpp
constexpr uint32_t FADEIN_MS = 900;
constexpr float ALTSIZE_LO = 1.35f, ALTSIZE_HI = 0.75f, ALTSIZE_CEIL_FT = 40000.0f;
```

- [ ] **Step 2 — Thread opacity + scale into the shape helpers.** `blip_fill_dsc(g, ac)` gains an opacity: set `g.bg_opa = opa` (and scale the outline opacity too). `draw_chevron/draw_silhouette/draw_diamond` and the inline kite path gain a `float scale` on their local points (multiply each local `(lx,ly)` by `scale` before `blip_rot`/offset; diamond scales `r`). Simplest: give the helpers `(d, ac, opa, scale)` signatures; the kite path multiplies `GX[i]*scale`/`GY[i]*scale` and sets its rect dsc `bg_opa = opa`. Default callers pass `255, 1.0f` where fx don't apply.

- [ ] **Step 3 — Compute per-aircraft opa + scale in `ac_draw_cb`** (non-orb path):
```cpp
    const uint32_t age = lv_tick_get() - ac.firstSeenMs;
    const lv_opa_t opa = (age >= FADEIN_MS) ? LV_OPA_COVER : (lv_opa_t)(255u * age / FADEIN_MS);
    float scale = 1.0f;
    if (s_desc->blipFx & FX_ALTSIZE) {
        float t = ac.altFt / ALTSIZE_CEIL_FT; if (t < 0) t = 0; if (t > 1) t = 1;
        scale = ALTSIZE_LO + (ALTSIZE_HI - ALTSIZE_LO) * t;
    }
```
Pass `opa`/`scale` into whichever shape draw runs. Also apply `opa` to the floating call/alt label descriptors (`lc.opa`/`la.opa`) so the whole blip fades as one. Orb path: apply `opa` to the ball (optional; keep simple — orb can pass 255 if it complicates).

- [ ] **Step 4 — Verify:** both builds SUCCESS. Code-trace: a just-appeared aircraft (`age≈0`) draws near-transparent and ramps to solid by 900ms; Mission Control glyphs scale 1.35→0.75 with altitude; every other theme uses `scale=1.0`; fully-faded aircraft on non-altsize themes get `opa=255, scale=1.0` → identical to before.

- [ ] **Step 5 — Commit:** `git add src/radar_view.cpp && git commit -m "feat(fx): fade-in opacity + altitude-scaled glyph size"`

## Task 4: Pulse + heading-vector + ATC leader line

**Files:** `src/radar_view.cpp` (`ac_draw_cb`)

- [ ] **Step 1 — Constants:**
```cpp
constexpr uint32_t PULSE_MS = 900;
constexpr float VEC_SCALE = 0.052f, VEC_MIN = 6.0f, VEC_MAX = 30.0f, VEC_MIN_KT = 40.0f;
```

- [ ] **Step 2 — Pulse (global, all themes)**, drawn BEFORE the glyph so it sits behind: if `age < PULSE_MS`, one expanding ring via `lv_draw_arc` at `ac.pos`, radius `4 + 18.0f*age/PULSE_MS`, opa `(lv_opa_t)(200 - 200*age/PULSE_MS)`, color `ac.color`. If `ac.emergency`, additionally a repeating red ring: phase `p = (s_frameCtr % 30)/30.0f`, radius `6 + 22*p`, opa `(lv_opa_t)((1-p)*220)`, color `COL_EMERG` (augments the existing static halo — leave that in place).

- [ ] **Step 3 — Heading-vector (FX_VECTOR)**, drawn near the glyph, only when `!ac.onGround && ac.gsKt >= VEC_MIN_KT`: length `len = clamp(ac.gsKt*VEC_SCALE, VEC_MIN, VEC_MAX)`; endpoint via the file's rotation (reuse `rim_point`-style or the `blip_rot` math with local point `(0, -len)` so it points along `track`); `lv_draw_line` from `ac.pos` to the endpoint, width 2, color `ac.color`, opa scaled by the fade `opa`.

- [ ] **Step 4 — ATC leader line (FX_LEADER)**, drawn with the labels (non-orb, where labels exist): a thin `lv_draw_line` width 1 from `ac.pos` to the label anchor (top-left of the existing `a1` call-label box, ~`(ac.pos.x+12, ac.pos.y-14)`), color `s_cSoft`, opa ~`120` scaled by fade `opa`. Draw it just before the label so the label sits on top.

- [ ] **Step 5 — Verify:** both builds SUCCESS. Code-trace: new/emergency contacts pulse; Military/CIC/ClaudeIC show a forward speed-vector + a connector to the label; non-fx themes are unchanged (all three gated on `blipFx` / global-but-only-when-new).
